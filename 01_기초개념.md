# Git / GitHub 핵심 개념

## Git vs GitHub

| | Git | GitHub |
|--|-----|--------|
| 정체 | 버전 관리 도구 | Git 저장소 호스팅 서비스 |
| 비유 | 카메라 (촬영 도구) | 사진첩 (보관 공간) |
| 설치 | 내 컴퓨터에 설치 | 웹 서비스 (브라우저로 접속) |

> ⚠️ GitHub는 코드를 **실행**하는 곳이 아니라 **저장**하는 곳이에요.

---

## 핵심 용어 6가지

### Repository (저장소)
프로젝트 폴더 전체. 코드와 변경 역사가 모두 담긴다. 줄여서 'repo'라고 부른다.

### Commit
변경 내역 저장. 게임의 세이브 포인트.

좋은 커밋 메시지 예시:
- ✅ `로그인 버튼 색상 수정`
- ✅ `README 설치 방법 추가`
- ❌ `수정함`
- ❌ `asdfgh`

### Branch
메인에서 뻗어나온 가지, 독립된 작업 공간.
메인 코드는 건드리지 않고 새 기능을 따로 개발할 수 있다.
잘 되면 합치고, 안 되면 버리면 그만.

### Fork
남의 저장소를 내 계정으로 복사.
원본에 영향 없이 자유롭게 수정 가능.

### Push / Pull
- **Push**: 내 컴퓨터 → GitHub 서버 (올리기)
- **Pull**: GitHub 서버 → 내 컴퓨터 (내려받기)

### Pull Request (PR)
Fork한 코드를 수정한 뒤 원작자에게 반영 요청.
오픈소스 기여의 핵심.

## 라이선스 기초

| 라이선스 | 핵심 한 줄 | 대표 사례 |
|----------|-----------|----------|
| **MIT** | 거의 뭐든 OK, 출처만 밝혀줘 | React, VS Code, Flask |
| **GPL** | 수정했으면 너도 공개해야 해 | Linux, **Git**, WordPress |
| **Apache 2.0** | MIT + 특허 보호 | Android, TensorFlow |

> 자세한 비교: [choosealicense.com](https://choosealicense.com)
