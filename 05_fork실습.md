# Fork 실습 가이드

## 이 저장소가 뭔가요?

여러분이 **첫 번째 Pull Request**를 경험해보기 위한 실습용 저장소예요.
아래 순서대로 따라하면 여러분의 이름이 이 저장소에 영원히 남습니다! 🎉

---

## 실습 순서

### 1단계 — Fork
오른쪽 상단 `Fork` 버튼 클릭 → 내 계정으로 복사

### 2단계 — 내 파일 만들기
내 Fork 저장소에서:
1. `Add file` → `Create new file`
2. 파일명: `contributors/내이름.md` (예: `contributors/홍길동.md`)
   - `contributors/` 를 앞에 붙이면 자동으로 폴더가 생겨요!
3. 아래 내용 입력:

```markdown
# 홍길동

한마디: "오픈소스의 거인이 될 거예요!"
```

### 3단계 — Commit
`Commit changes` → 메시지 입력 → `Commit changes`

### 4단계 — Pull Request
`Contribute` → `Open Pull Request` → `Create Pull Request`

### 5단계 — 기다리기 🙏
선생님이 merge해주면 내 이름이 공식 저장소에 올라가요!

---

## 💡 왜 파일을 따로 만드나요?

같은 파일을 여러 명이 동시에 수정하면 **충돌(Conflict)** 이 생겨요.
각자 다른 파일을 만들면 충돌 없이 모두의 PR이 깔끔하게 merge됩니다.
실제 오픈소스에서도 번역 파일이나 언어별 문서를 이렇게 기여해요!

---

## 💡 이게 바로 오픈소스 기여예요

**Fork → 수정 → PR → Merge!**

전 세계 오픈소스 프로젝트에 기여하는 방법이 딱 이겁니다.
